### 消失的密钥 (The Vanishing Key)

#### 题目信息
- 题目名称：消失的密钥 (The Vanishing Key)
- 题目类型：Web
- 题目分值：50分
- 题目地址：39.105.213.28:12601

#### 题目描述
实验室的防护系统似乎有些老旧，它对"Key"这个词异常敏感。如果你能证明你比它的过滤器更聪明，或许能看到隐藏在迷雾后的真相。

#### 解题过程

##### 阶段一：绕过过滤器
通过测试发现：
- 过滤器会删除所有小写的"key"子串
- 大写的"Key"或"KEY"不会被删除，但会被验证逻辑拒绝
- 我们需要输入一个字符串，经过过滤后得到"key"

构造 payload：`kkeyey`

原理：`kkeyey` 中包含 `key`（第2-4位），删除后剩下 `k` + `ey` = `key`

成功！第一阶段通过！

##### 阶段二：POST数据
第一阶段成功后，页面提示需要POST数据结构'a'。

通过测试发现：
- 需要发送 `a[key]=1337` 作为 form-data（注意是字符串 "1337"）
- 成功通过第二阶段！

##### 阶段三：哈希碰撞
第二阶段成功后，页面提示：
```
[SYS] Collision check pending. Collision pending. (Params 'a', 'b' required via GET.)
```

现在需要通过 GET 参数提供 a 和 b，它们应该是哈希碰撞对！

由于题目提示是老旧系统，我们尝试了 PHP 弱类型 MD5 碰撞：

**原理：**
如果两个字符串的 MD5 哈希值都是 "0e..." 开头，PHP 会把它们当作 0 处理，所以 == 比较结果为 True。

**构造 payload：**
- a = `QNKCDZO`，MD5 = `0e830400451993494058024219903391`
- b = `240610708`，MD5 = `0e462097431906509019562988736854`

成功通过第三阶段！

#### 最终 Flag
`ISCC{QN-tGwW0yZD4!1fQ?TXJ0b0)bUag8i}`
