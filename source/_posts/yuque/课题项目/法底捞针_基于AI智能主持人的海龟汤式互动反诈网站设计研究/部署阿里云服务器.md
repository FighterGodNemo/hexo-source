ssh -p 22 root@47.93.149.132 "cd /var/www && tar --exclude='project/frontend/node_modules' --exclude='project/backend/node_modules' --exclude='project/frontend/build' -czf project-src.tar.gz project"

2. 本地下载（按你指定目录）

scp -P 22 root@47.93.149.132:/var/www/project-src.tar.gz "C:\Users\glj07\Desktop\法底捞针：基于AI智能主持人的海龟汤式互动反诈网站设计研究\project-src.tar.gz"



1. 在本机启动前端开发服务器（支持热更新）

cd C:\Users\glj07\Desktop\法底捞针：基于AI智能主持人的海龟汤式互动反诈网站设计研究\已部署到服务器的项目\第二版\frontend  
cp .env.example .env  
npm install  
npm start

2. 浏览器打开 [http://localhost:3007，改动](http://localhost:3007，改动) src/ 下的颜色样式会秒级生效。

  注意两点

+ 你改了 tailwind.config.js 或 .env 时，通常要重启 npm start 才会生效。
+ 如果页面要请求后端数据，你有两个选择：
    1. 本地起后端（最稳）cd C:\Users\glj07\Desktop\法底捞针：基于AI智能主持人的海龟汤式互动反诈网站设计研究\已部署到服务器的项目\第二版\backend  
cp .env.example .env  
npm install  
npm run dev  
然后在前端 .env 里设：REACT_APP_API_URL=[http://localhost:3008](http://localhost:3008)
    2. 直接连线上后端（更省事，但可能被 CORS 拦截）  
需要后端 CORS_ORIGIN 允许 [http://localhost:3007，否则浏览器会报跨域。](http://localhost:3007，否则浏览器会报跨域。)

  如果你想看“线上实际效果”

+ 改完前端后在服务器上 npm run build，Nginx 静态文件更新后刷新即可。这个比本地热更新慢，但能保证跟线上一致。
